# Отчёт о проекте Audio Stem Separator

## Выбор модели
В качестве модели разделения использована предобученная архитектура **htdemucs** из библиотеки Demucs.  
Модель сочетает Transformer и U-Net, обеспечивая хороший баланс качества и скорости.  
Достоинства:  
- Высокое качество разделения (SDR на MUSDB18 ~5–10 дБ)  
- Работает на CPU и GPU   

## Техническая реализация
- **Бэкенд**: FastAPI + Demucs. Эндпоинт `/separate` принимает аудиофайл и возвращает ZIP-архив с 4 стемами (vocals, drums, bass, other). CORS настроен для доступа с фронтенда.  
- **Фронтенд**: React‑приложение, отправляющее файл на сервер и отображающее стемы с аудиоплеерами и возможностью скачивания по отдельности или общим архивом.  
- **Оценка качества**: скрипт `evaluate.py` вычисляет метрики SDR, SIR, SAR с помощью `mir_eval` на тестовом наборе MUSDB18 (50 треков).

## Достигнутое качество
При изолированной оценке (каждый стем сравнивается с эталоном отдельно) получены следующие средние значения:

| Stem   | SDR, дБ | SIR, дБ | SAR, дБ |
|--------|---------|---------|---------|
| bass   | 8.77    | inf     | 8.77    |
| drums  | 10.67   | inf     | 10.67   |
| other  | 5.07    | inf     | 5.07    |
| vocals | 8.59    | inf     | 8.59    |

**SIR = inf** объясняется методологией расчёта: при подаче единственного источника функция `mir_eval.bss_eval_sources` не имеет информации о других стемах, и проекция помех (`e_interf`) равна нулю.

Вывод: модель успешно разделяет инструменты, особенно хорошо выделяются ударные (10.67 дБ) и вокал. Качество для «other» (аккомпанемент) ниже, что типично для сложных смешанных сигналов.

## Узкие места и возможные улучшения
- **Качество для сложных жанров**: модель может ошибаться на плотных миксах, где инструменты сильно перекрываются по частотам.
- **Производительность**: на CPU обработка длинного трека занимает длительное время.
- **API**: добавление поддержки потоковой загрузки больших файлов и индикации прогресса.

## Заключение
Проект представляет собой рабочий прототип сервиса разделения аудио с веб-интерфейсом и количественной оценкой качества. Модель Demucs (htdemucs) показала приемлемые результаты, пригодные для многих практических задач.
